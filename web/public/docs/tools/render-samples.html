Render Samples¶

The render-samples script orchestrates the rendering of sample project variants and runs module smoke tests.

Purpose¶

  • Render sample projects from answer files

  • Execute module-specific smoke tests

  • Generate render metadata and timing reports

  • Validate Dockerfile configurations

Location¶

scripts/render-samples.sh

Usage¶

Render Default Sample¶

./scripts/render-samples.sh
# or via Makefile:
make samples

Render Specific Variant¶

./scripts/render-samples.sh \
  --variant my-variant \
  --answers samples/my-variant/copier-answers.yml

With Container Validation¶

./scripts/render-samples.sh --validate-containers

Features¶

Module Smoke Tests¶

The script runs smoke tests for enabled modules:

Module

Test Command

CLI

pytest tests/test_cli.py

Python API

pytest tests/test_api_fastapi.py

Node API

pnpm --filter api-node test

Docs (Fumadocs)

pnpm --filter docs-fumadocs build

Docs (Sphinx)

uv run make linkcheck

Quality (make)

make quality

Quality (uv)

uv run task quality

Output Artifacts¶

Each render produces:

  • metadata.json - Render timing and configuration

  • smoke-results.json - Test results per module

  • doc-publish.json - Documentation publish status

Custom Copier Command¶

Override the copier command for CI environments:

COPIER_CMD=/path/to/copier ./scripts/render-samples.sh

Sample Structure¶

samples/
├── default/
│   ├── copier-answers.yml
│   ├── render/           # Generated output
│   ├── metadata.json
│   └── smoke-results.json
└── my-variant/
    └── copier-answers.yml
