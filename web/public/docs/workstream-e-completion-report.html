Copier.yml Integration¶

build_tool:
  type: str
  help: "Primary build/bundling tool"
  choices:
    - vite
    - webpack
    - parcel
    - esbuild
    - rollup
    - turbopack
    - rspack
    - farm
  default: vite

  type: str
  help: "Monorepo management tool"
  choices:
    - none
    - turborepo
    - nx
    - rush
    - lerna
    - moon
    - bazel
  default: none

Hook Integration¶

Add to template/hooks/pre_gen_project.py:

from validators.build_tool_validator import validate_build_tool
from validators.monorepo_tool_validator import validate_monorepo_tool

# Build tool validation
build_valid, build_summary = validate_build_tool(config)
print(build_summary)
if not build_valid:
    raise ValueError("Build tool validation failed")

# Monorepo tool validation
monorepo_valid, monorepo_summary = validate_monorepo_tool(config)
print(monorepo_summary)
if not monorepo_valid:
    raise ValueError("Monorepo tool validation failed")

Future Enhancements¶

Potential Additions¶

  1. Additional Build Tools:

    • SWC (standalone bundler mode)

    • Bun bundler (native)

    • Rome/Biome toolchain

  2. Enhanced Configurations:

    • PWA support in build configs

    • Web Workers configuration

    • Service Worker templates

    • WASM bundling support

  3. Monorepo Enhancements:

    • Rush monorepo tool support

    • Pnpm workspaces without tools

    • Yarn PnP configurations

    • Remote caching setup guides

  4. Validation Improvements:

    • Runtime dependency checking

    • Version compatibility validation

    • Plugin ecosystem checking

    • Performance recommendations

Known Limitations¶

  1. Build Tools:

    • Turbopack is Next.js exclusive (by design)

    • esbuild plugin ecosystem is limited

    • Parcel v3 breaking changes may require updates

  2. Monorepo Tools:

    • Bazel has steep learning curve

    • Lerna lacks caching (by design)

    • Moon is newer with smaller ecosystem

  3. Validators:

    • Cannot check actual tool availability

    • Cannot validate tool versions

    • Warnings are recommendations, not errors

Testing Recommendations¶

Manual Testing Checklist¶

  • Render template with each build tool

  • Verify generated config files are valid

  • Test dev server startup for each tool

  • Test production builds

  • Verify monorepo configs with multiple packages

  • Test validators with invalid configurations

  • Check documentation links and examples

Automated Testing¶

# Run validator tests
uv run pytest tests/test_workstream_e_validators.py -v

# Test all build tool renders
for tool in vite webpack esbuild rollup rspack turbopack farm parcel; do
  echo "Testing $tool..."
  # Render with specific build tool
  copier copy . test-output --data build_tool=$tool
done

# Test all monorepo tool renders
for tool in turborepo nx lerna moon bazel; do
  echo "Testing $tool..."
  copier copy . test-output --data monorepo_tool=$tool
done
