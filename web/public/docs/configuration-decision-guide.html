Configuration Decision Guide¶

This guide helps you choose the right Riso template configuration for your project using visual decision trees and clear recommendations.

Quick Decision Trees¶

1. Backend Language Selection¶

Recommendations:

  • Python + FastAPI: Best for data science, ML, modern async APIs (most popular choice)

  • Node.js + Fastify: Best for real-time features, unified JS/TS stack

  • Rust + Axum: Best for performance-critical services, maximum type safety

  • Go + Gin: Best for operational simplicity, excellent concurrency

  • Java + Spring Boot: Best for enterprise, mature ecosystem

2. Frontend Framework Selection¶

Recommendations:

  • React + Vite: Best default choice, huge ecosystem, great tooling

  • Next.js: Best for SEO, server-side rendering, full-stack apps

  • Vue + Vite: Best for progressive enhancement, gentle learning curve

  • Svelte + SvelteKit: Best for small bundles, elegant syntax

  • Astro: Best for content-heavy marketing sites

3. Repository Structure Selection¶

Recommendations:

  • flat: Simple scripts, prototypes, learning projects

  • src-layout: Python packages, libraries to publish on PyPI

  • apps-packages: Monorepos with 2-5 services/apps

  • multi-workspace: Microservices, 6+ loosely coupled services

  • domain-driven: Enterprise apps with complex business domains

4. Component Library Selection¶

Recommendations:

  • shadcn/ui: Best for Tailwind users, full customization, no dependencies

  • MUI: Best for Material Design, comprehensive components

  • Chakra UI: Best for accessibility, dark mode, composability

  • Ant Design: Best for enterprise dashboards, data-heavy UIs

  • Mantine: Best for feature-rich needs (forms, notifications, editor)

  • Headless UI: Best for complete styling control, accessibility

5. Monorepo Tool Selection¶

Recommendations:

  • Turborepo: Best for TypeScript monorepos, simple setup, fast

  • Nx: Best for enterprise, powerful features, polyglot support

  • Lerna: Best for package versioning, mature ecosystem

  • Moon: Best for Rust-based speed, modern features

  • Bazel: Best for Google-scale, polyglot, 10+ services

6. Testing Strategy Selection¶

Recommendations:

  • Python: pytest (unit), Supertest (API), Playwright (E2E)

  • Node.js: Vitest/Jest (unit), Supertest (API), Playwright (E2E)

  • Frontend: Vitest (unit), Playwright (E2E), Storybook (component)

  • Full-Stack: Combine all above based on stack

Common Configuration Patterns¶

Pattern 1: Modern Full-Stack Startup¶

# copier-answers.yml
backend_languages: [python]
python_api_framework: fastapi
frontend_framework: react
react_framework: nextjs
react_state_management: zustand
component_library: shadcn-ui
css_framework: tailwind
repository_structure: apps-packages
monorepo_tool: turborepo
database_engine: postgres
containerization: enabled
docs_site: fumadocs

Use Case: SaaS product, 2-5 person team, rapid iteration

Pattern 2: Enterprise Microservices¶

backend_languages: [python, java]
python_api_framework: fastapi
java_framework: spring-boot
frontend_framework: react
react_framework: nextjs
component_library: ant-design
repository_structure: multi-workspace
monorepo_tool: nx
database_engine: postgres
containerization: enabled
monitoring_tools: opentelemetry

Use Case: Large organization, multiple teams, complex domain

Pattern 3: High-Performance Real-Time¶

backend_languages: [rust, node]
rust_web_framework: axum
node_api_framework: fastify
frontend_framework: svelte
svelte_framework: sveltekit
css_framework: tailwind
repository_structure: apps-packages
monorepo_tool: turborepo
websocket_module: enabled
containerization: enabled

Use Case: Real-time features, gaming, trading platforms

Pattern 4: Content-Heavy Marketing¶

Migration Paths¶

Adding Frontend to Backend-Only Project¶

Before:

backend_languages: [python]
frontend_framework: none

After:

backend_languages: [python]
frontend_framework: react
react_framework: vite

Steps:

  1. Re-render with updated configuration

  2. Install Node.js dependencies: pnpm install

  3. Start dev servers: make dev-all

  4. Test integration endpoints

Converting to Monorepo¶

Before:

repository_structure: src-layout
monorepo_tool: none

After:

repository_structure: apps-packages
monorepo_tool: turborepo

Steps:

  1. Create backup: git checkout -b backup

  2. Reorganize into apps/ and packages/

  3. Add turbo.json configuration

  4. Update import paths

  5. Test: pnpm turbo build test

Adding Another Backend Language¶

Before:

backend_languages: [python]

After:

backend_languages: [python, rust]
rust_web_framework: axum

Steps:

  1. Re-render or manually add Rust structure

  2. Set up Rust toolchain: rustup default stable

  3. Configure cross-language communication (gRPC/REST)

  4. Update CI workflows for Rust testing

Troubleshooting¶

Issue: Frontend build fails¶

Solution:

# Clear caches
rm -rf node_modules .pnpm-store dist
pnpm install --frozen-lockfile
pnpm build

Issue: Python package not found¶

Solution:

# Reinstall in editable mode
uv pip install -e .
# Or sync dependencies
uv sync

Issue: Monorepo tasks fail¶

Solution:

# Turborepo
pnpm turbo build --force

# Nx
pnpm nx reset
pnpm nx run-many -t build

Issue: Container health check fails¶

Solution:

# Check logs
docker logs <container-name>

# Verify health endpoint
docker exec <container-name> curl http://localhost:8000/health

# Rebuild without cache
docker build --no-cache -t app:latest .
