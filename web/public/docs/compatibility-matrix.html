Decision Trees¶

Use these decision trees to guide your configuration choices:

Alternative Recommendations¶

Backend Alternatives¶

FastAPI: Consider Flask for simpler APIs, Django for CMS/admin needs

Rust: Consider Go for easier learning curve with good performance

Node.js: Consider Bun for even faster execution (experimental)

Frontend Alternatives¶

React: Consider Preact for smaller bundle size, Solid for fine-grained reactivity

Next.js: Consider Remix for web standards focus, Astro for content sites

Tailwind: Consider UnoCSS for faster builds, Panda CSS for type-safe styles

Tooling Alternatives¶

Turborepo: Consider Nx for more features, Moon for Rust-based speed

Vite: Consider Turbopack (Next.js), esbuild for maximum speed

pnpm: Consider Bun for all-in-one toolchain, npm for simplicity

Best Practices¶

Backend Selection¶

  • Choose Python for data science/ML integrations

  • Choose Rust for maximum performance and type safety

  • Choose Node.js for real-time features and npm ecosystem

  • Choose Go for operational simplicity and deployment ease

Frontend Selection¶

  • Start with Vite + React for most projects

  • Use Next.js for SEO-critical sites or full-stack apps

  • Choose Svelte for smaller bundle sizes

  • Pair shadcn/ui with Tailwind for customizable components

Repository Structure¶

  • Use src-layout for single-package Python projects

  • Use apps-packages for TypeScript monorepos

  • Use domain-driven for complex business domains

  • Avoid monorepo for simple single-service projects

Testing Strategy¶

  • Use pytest for Python, Vitest for TypeScript

  • Add Playwright for critical user flows

  • Enable coverage reporting in CI

  • Run tests on matrix of Python/Node versions

Incompatible Combinations¶

The following combinations are not supported and will fail validation:

frontend_framework¶

  • frontend_framework=none + component_library → Component libraries require a frontend framework

  • frontend_framework=none + css_framework → CSS frameworks require a frontend framework

  • frontend_framework=none + icon_library → Icon libraries require a frontend framework

project_layout¶

  • project_layout=single-package + monorepo_tool → Monorepo tools only apply to monorepo layouts

backend_languages¶

  • backend_languages=[] + python_api_framework → Python framework requires Python in backend_languages

  • backend_languages=[] + rust_web_framework → Rust framework requires Rust in backend_languages

  • backend_languages=[] + node_api_framework → Node framework requires Node in backend_languages
