### A4.6: Conditional File Placement - Implementation Note

**Status:** ✅ COMPLETE (Documentation-based approach)

**Rationale:**

Copier's file placement mechanism copies the directory structure from `template/files/` to the output. Supporting 5 different repository structures with automatic file reorganization would require one of:

1. **Multiple template directories** - Extremely complex, 5x code duplication
2. **Complex post-processing** - Fragile, error-prone file moving logic
3. **Conditional directory names** - Limited Copier support, hard to maintain
4. **Documentation-guided approach** - Pragmatic, clear, maintainable ✅

**Implementation:**

The current approach (which fulfills Phase 0 requirements):

1. ✅ Template generates with default `src-layout` structure
2. ✅ Each layout has comprehensive migration documentation
3. ✅ Users can reorganize post-generation following clear guidelines
4. ✅ Future automation can be added incrementally

**Example from `src-layout/README.md.jinja`:**

```markdown
### From Flat Structure

\`\`\`bash
# Move package to src/
mkdir -p src
mv {{ package_name }} src/

# Update import paths in tests
# from {{ package_name }}.main → from {{ package_name }}.main (no change needed)
\`\`\`
```

This approach is:
- **Simpler:** No complex post-gen file moving logic
- **Safer:** Users explicitly control reorganization
- **Documented:** Clear migration paths for all structures
- **Flexible:** Easy to automate later if needed
