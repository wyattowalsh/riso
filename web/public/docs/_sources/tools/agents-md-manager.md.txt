# agents-md-manager

```{rubric} AGENTS.md Management Skill
```

```{admonition} Overview
:class: tip

AI agent skill for managing AGENTS.md as the single source of truth (SSOT) across AI coding platforms including Claude Code, GitHub Copilot, Cursor, Amazon Q, and Gemini.
```

## Quick Reference

| Property | Value |
|----------|-------|
| **Category** | AI Skills |
| **Type** | Automation |
| **Location** | `.claude/skills/agents-md-manager/` |
| **Platforms** | Claude, Copilot, Cursor, Codex, Amazon Q, Gemini |

## Features

- **Detection** — Scan codebase for all agent instruction files
- **Analysis** — Score AGENTS.md quality against GitHub's 6-area framework
- **Sync** — One-way sync from AGENTS.md to platform pointer files
- **Migration** — Convert deprecated formats (e.g., `.cursorrules` → `.cursor/rules/`)

## Usage

### Detect Agent Files

```bash
uv run python .claude/skills/agents-md-manager/detect.py /path/to/repo
```

Output includes:
- **present**: Files found with location and type
- **missing**: Expected files not found
- **deprecated**: Legacy formats requiring migration
- **nested**: Subdirectory-level files (monorepo support)

### Analyze Quality

```bash
uv run python .claude/skills/agents-md-manager/analyze.py AGENTS.md
```

Scores against GitHub's 6-area framework:
1. Commands (build/test/lint with flags)
2. Testing (framework, patterns, coverage)
3. Project structure (key directories, entry points)
4. Code style (language, formatting, conventions)
5. Git workflow (branching, commits, PRs)
6. Boundaries (never touch, ask first, always do)

### Sync to Platforms

```bash
# Sync to all platform pointer files
uv run python .claude/skills/agents-md-manager/sync.py /path/to/repo

# Generate missing files only
uv run python .claude/skills/agents-md-manager/sync.py /path/to/repo --missing-only
```

### Migrate Formats

```bash
uv run python .claude/skills/agents-md-manager/migrate.py /path/to/repo
```

Handles:
- `.cursorrules` → `.cursor/rules/main.mdc`
- Legacy `CLAUDE.md` content → AGENTS.md consolidation
- Inline instructions → pointer file conversion

## Best Practices

From analysis of 2,500+ repositories:

1. **Commands first** — Put executable commands in first 20 lines
2. **Concrete examples** — One code snippet beats three paragraphs
3. **Explicit boundaries** — Define "never touch" items
4. **Stack-specific** — "React 18 + TypeScript + Vite" not just "React project"
5. **Six areas covered** — Commands, testing, structure, style, git, boundaries
