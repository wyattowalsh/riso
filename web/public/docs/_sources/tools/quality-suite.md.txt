# Quality Suite

The quality suite is a unified Makefile target that runs the complete code quality pipeline.

## Purpose

- Run all quality checks in optimal order
- Fail fast on first error
- Provide consistent CI and local development experience
- Aggregate quality tool outputs

## Usage

### Run Full Suite

```bash
make quality
```

### Individual Components

```bash
make lint        # Ruff linting and formatting check
make typecheck   # ty type checking
make test        # pytest execution
```

### With Coverage

```bash
make test-cov    # pytest with coverage report
```

## Pipeline Stages

```
make quality
    │
    ├─→ make lint
    │     ├─→ ruff check
    │     └─→ ruff format --check
    │
    ├─→ make typecheck
    │     └─→ ty check
    │
    └─→ make test
          └─→ pytest
```

## Makefile Definition

```makefile
.PHONY: quality
quality: lint typecheck test ## Run full quality suite
    @printf "$(GREEN)✓ All quality checks passed$(RESET)\n"

.PHONY: lint
lint: ## Run ruff linter
    uv run ruff check scripts template/hooks tests
    uv run ruff format --check scripts template/hooks tests

.PHONY: typecheck
typecheck: ## Run ty type checker
    uv run ty check scripts template/hooks

.PHONY: test
test: ## Run pytest
    uv run pytest tests -v
```

## CI Integration

The quality suite is designed for CI usage:

```yaml
# .github/workflows/ci.yml
- name: Run quality checks
  run: make ci  # Runs install + quality

- name: Verify documentation
  run: make ci-docs  # Runs docs-build + linkcheck
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | All checks passed |
| 1 | Lint failure |
| 2 | Type check failure |
| 3 | Test failure |
